# 优先级规则

## 优先级枚举

- `urgent`：需要当天处理，存在明确截止时间、客户升级、付款或重大风险。
- `high`：需要优先处理，但没有立即风险。
- `normal`：常规业务沟通。
- `low`：参考信息、营销信息或无需明确动作。

## 判定信号

### 提高优先级

- 出现明确截止时间。
- 涉及付款延迟、合同风险、交付延期或客户投诉。
- 发件人要求立即回复。
- 邮件来自关键客户或管理层。

### 降低优先级

- 无明确动作要求。
- 仅为抄送或通知。
- 内容属于广告、资讯或非业务相关。

## 输出要求

优先级必须附带一句理由，说明依据来自邮件中的哪些信号。
