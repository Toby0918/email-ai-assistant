# 路线图

## 阶段 1：本地原型

- 完成 Python 后端分析 API。
- 完成当前邮件输入格式和分析结果 JSON schema。
- 完成邮件正文清洗、AI 调用、风险识别和回复草稿生成。
- 使用本地调试页面或模拟前端验证“点击分析当前邮件”。
- 不接入真实邮箱账号。

## 阶段 2：辅助窗口路线选择

- 评估 Outlook Add-in、浏览器扩展和 Google Workspace Add-on。
- 明确目标企业邮箱环境。
- 确定前端框架和构建工具。
- 继续禁止前端保存 API key。

## 阶段 3：受控邮箱集成

- 在单独确认后接入测试邮箱或企业沙箱账号。
- 验证当前邮件识别能力。
- 加入最小权限授权。
- 审计日志和隐私处理规则。

## 阶段 4：团队协作能力

- 可配置分类和优先级规则。
- 可配置回复风格。
- 人工审核后的草稿插入。
- 管理员可见的安全策略。
