# 用户流程

## 第一阶段主流程

1. 用户在企业邮箱中打开一封邮件。
2. 辅助窗口读取当前邮件的主题、发件人、收件人、时间和正文。
3. 用户点击“分析此邮件”。
4. 前端把当前邮件内容提交给 Python 后端。
5. 后端清洗正文并调用 AI 模型。
6. 后端返回结构化分析结果。
7. 辅助窗口展示分析结果。
8. 用户人工确认后复制或改写回复草稿。

## 异常流程

- 当前邮件无法识别：提示用户刷新邮件或重新打开邮件。
- 正文为空：提示无法分析空邮件，并允许用户手动粘贴测试内容。
- AI 返回不可解析 JSON：后端返回错误码，前端提示稍后重试。
- 后端不可用：前端提示本地分析服务未启动。

## 禁止流程

- 用户未点击按钮时自动分析所有邮件。
- 自动把回复草稿发送出去。
- 自动删除或归档邮件。
- 未确认前接入真实邮箱账号。
